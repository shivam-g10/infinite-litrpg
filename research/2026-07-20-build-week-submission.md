# Build Week Submission Requirements

- Researched: 2026-07-20
- Scope: OpenAI Build Week Challenge submission evidence

## Facts

- Submission closes July 21, 2026 at 5:00 PM PDT, which is July 22, 2026 at 05:30 IST.
- Entry needs a working project built with Codex and GPT-5.6 in one declared track.
- Submission needs an English project description explaining features and functionality.
- Demo must be publicly visible on YouTube, under three minutes, include audio, show the working project, and explain both Codex and GPT-5.6 use.
- Repository must be public with a relevant license, or private and shared with `testing@devpost.com` and `build-week-event@openai.com`.
- README must include setup and testing guidance, explain Codex collaboration and acceleration, identify key decisions, and explain GPT-5.6 use.
- Submission needs the `/feedback` Codex Session ID from the primary build task.
- Judges may evaluate only the description, images, and video. They are not required to install or test the project.

## Inference

- Submission packet must be understandable without running local code.
- Demo script must reserve spoken time for Codex and GPT-5.6 evidence.
- Repository evidence should include current screenshots, architecture, one complete trace, exact verification results, and a short testing path.
- User-controlled URLs and session ID should remain explicit release gates rather than placeholders presented as complete.

## Unknowns

- Final public YouTube URL.
- Final public repository state or private judge-sharing confirmation.
- Primary `/feedback` Session ID.
- Final twelve-cycle report and six narrative review approvals.
- User authorization to publish and submit.

## Decision Impact

- Add a paste-ready submission packet and timed demo script.
- Add required Codex collaboration section to README.
- Keep user-owned publication fields visibly incomplete.
- Do not claim submission-ready until final live and narrative gates pass.
